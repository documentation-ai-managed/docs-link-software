## Document Organization and Structuring

You create hierarchical structures to keep your documentation intuitive and scalable. Start with spaces for projects, add folders for categories, and nest pages for details.

<Steps>
  <Step title="Create a Space" icon="plus">
    Navigate to your dashboard and click `New Space`. Enter a name like `Project Alpha Docs`.
  </Step>
  <Step title="Add Folders" icon="folder-plus">
    Inside the space, create folders such as `Getting Started` and `API Reference`.
  </Step>
  <Step title="Build Pages" icon="file-text">
    Add pages with rich text, embeds, and code blocks. Use templates for consistency.
  </Step>
</Steps>

<Callout kind="tip">
  Use templates to standardize page layouts across your team.
</Callout>

## Version History and Tracking

Every edit creates a version snapshot. You compare changes, restore previous versions, and track who made updates.

<CodeGroup tabs="Web,CLI">
  ```javascript
  // Fetch version history via API
  const history = await fetch('https://api.example.com/spaces/proj-alpha/pages/page-456/versions');
  const versions = await history.json();
  console.log(versions); // Array of {id, author, timestamp, changes}
  ```
  ```bash
  # CLI command to revert
  link-software versions list --space proj-alpha --page page-456
  link-software versions revert --version v3.2.1
  ```
</CodeGroup>

## Advanced Search Functionalities

Search across all your spaces with filters for content type, author, or date. Use Boolean operators for precision.

<Expandable title="Search Examples" default-open="true">
  - Basic: `API authentication`
  - Filtered: `from:john@company.com setup`
  - Advanced: `"error handling" AND (React OR Vue)`

  ```javascript
  // API search query
  const results = await fetch('https://api.example.com/search?q=API+auth&filter=pages&space=proj-alpha');
  ```
</Expandable>

<Callout kind="info">
